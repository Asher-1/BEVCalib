BEVCalib Training Framework

Multi-Layer Script Chain + DDP + AMP + Adaptive Scheduling — Deep Analysis
训练框架设计哲学: 四层脚本链 (YAML → batch_train.sh → start_training.sh → train_universal.sh → train_kitti.py) 实现了配置与执行的完全分离。YAML 定义实验，Shell 脚本处理环境和分布式设置，Python 专注训练逻辑。 这种分层设计使得单实验调试、批量实验、多节点分布式训练可以复用同一套训练代码。

Training Loop — 单步训练详解
🔄

train_one_epoch() — 每个 training step 的完整流程

1. Data Prep
generate_perturbation()
→ init_T from gt_T
color_jitter (optional)
pc_jitter/dropout (optional)
→ move to device
→
2. Forward AMP
optimizer.zero_grad(set_to_none=True)
with autocast(fp16):
  T_pred, init_loss, loss =
    model(imgs, pcs, gt_T, init_T,
      cam2ego, intrinsic, masks)
→
3. Backward AMP
scaler.scale(loss).backward()
scaler.unscale_(optimizer)
clip_grad_norm_(max=35.0)
scaler.step(optimizer)
scaler.update()
→
4. Metrics
compute_batch_pose_errors()
→ trans/rot/roll/pitch/yaw
→ fwd/lat/ht errors
accumulate epoch_pose_errors
TensorBoard: Loss + Projection
关键细节: set_to_none=True 比 zero_grad() 更高效（避免内存分配）。 梯度裁剪 max_norm=35.0 在 unscale_ 之后执行，确保裁剪在真实梯度尺度上进行。 AMP 使用 FP16 + GradScaler，对 Deformable Attention 的数值稳定性至关重要。
Optimizer & Scheduler — 学习率策略
📈

AdamW Optimizer Optimizer

双参数组策略:
backbone_params = [img_branch, pc_branch]
head_params = [fuser, transformer, heads]

optimizer = AdamW([
  {'params': backbone_params, 'lr': lr * backbone_lr_scale},
  {'params': head_params, 'lr': lr},
], weight_decay=1e-4)
backbone_lr_scale = 0.1 (默认)
Backbone 使用 1/10 学习率，保护预训练权重
设计思考: SwinT-Tiny 和 SparseEncoder 已有良好的特征提取能力， 过大的学习率会破坏预训练表示。0.1 倍率实现了"近似冻结但允许微调"的效果。
📈

LR Scheduler Scheduler

cosine_warm_restarts (推荐):
SequentialLR([
  LinearLR(start_factor=0.01, total_iters=warmup_epochs=5),
  CosineAnnealingWarmRestarts(
    T_0=50, T_mult=2
  )
], milestones=[warmup_epochs])
step (备选):
StepLR(step_size=80, gamma=0.5)
Warm Restarts 优势: T_mult=2 使每次重启周期翻倍 (50→100→200 epochs)。 前期快速探索，后期精细收敛。Warmup 5 epochs 避免初始大梯度破坏预训练权重。
Distributed Training — DDP + AMP
🌐

DDP (DistributedDataParallel) DDP

初始化:
dist.init_process_group(
  backend='nccl',
  timeout=timedelta(minutes=30)
)
model = DDP(model,
  device_ids=[local_rank],
  find_unused_parameters=True
)
多节点支持:
单节点: torchrun --standalone --nproc_per_node=N
多节点: torchrun --nnodes=M --node_rank=R
  --rdzv_backend=c10d --rdzv_endpoint=MASTER:PORT
SLURM: 自动检测 SLURM_NNODES, SLURM_NODEID
K8s: MASTER_ADDR, RANK, WORLD_SIZE from env
find_unused_parameters=True: 因为 BEVEncoder 是可选模块， 某些配置下部分参数不参与前向传播。这会带来少量性能开销但确保正确性。
⚡

AMP (Automatic Mixed Precision) AMP

配置:
amp_dtype = torch.float16
scaler = GradScaler(enabled=True)
训练步骤中的 AMP 流程:
1. autocast(fp16) 包裹 forward pass
2. scaler.scale(loss).backward()
3. scaler.unscale_(optimizer)
4. clip_grad_norm_(max=35.0)
5. scaler.step(optimizer)
6. scaler.update()
为什么用 FP16 而非 BF16: 兼容更多 GPU 架构 (V100/A100/消费级)。 GradScaler 自动处理 FP16 的数值溢出问题。Deformable Attention 中的小数值运算 在 FP16 下需要 scaler 保护。
Validation & Early Stopping — 验证与早停
📊

Validation Loop

触发条件: epoch % eval_epoches == 0 (默认 20)

验证流程:
1. raw_model.eval() + torch.no_grad()
2. 遍历 val_loader (同 train 的 forward)
3. 收集 val_loss + val_pose_errors
4. TensorBoard: Loss/val/*, Epoch/val/*, Val/Projection
5. 更新 best_val 记录
验证指标 (8维):
trans_error, fwd_error, lat_error, ht_error
rot_error, roll_error, pitch_error, yaw_error
⚠

Early Stopping & Checkpoint

Early Stopping:
patience = early_stopping_patience (默认 0 = 禁用)
counter++ when val 不改善
counter ≥ patience → 停止训练
DDP: dist.broadcast(stop_tensor, src=0) 同步
Checkpoint 保存:
周期: ckpt_{epoch}.pth (每 40 epochs)
最优: ckpt_best_val.pth (val 改善时)
内容: model/optimizer state, errors,
  best_train/val, args, rotation_only
附加: 保存时运行 eval → projections + errors

Training Optimization Insights — 训练优化洞察
💡

训练框架优化方向分析

1. 梯度裁剪阈值
max_norm=35.0 较为宽松 → 可尝试自适应裁剪 或基于梯度统计动态调整阈值。
2. Warmup 策略
仅 5 epochs warmup → 对于 400 epoch 训练可能不够。 可尝试更长 warmup 或 warmup + cosine 的比例调整。
3. find_unused_parameters
=True 有性能开销 → 可通过条件编译 在确定配置下设为 False 提升 DDP 效率。
4. 验证频率
每 20 epochs 验证一次 → 可能错过最优点。 可尝试前期稀疏后期密集的验证策略。
5. EMA (Exponential Moving Average)
当前无 EMA → 可引入模型权重 EMA 提升验证和推理的稳定性。
6. 混合精度升级
FP16 → 在 A100/H100 上可升级为BF16， 无需 GradScaler，数值更稳定。
