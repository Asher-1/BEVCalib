BEVCalib Data Flow

Complete Tensor Shape Transformation Chain — From Raw Data to Calibration Output
数据流核心路径: 磁盘文件 → Dataset → collate_fn (crop/resize/pad/filter) → GPU → 双分支 BEV 投影 → 融合 → Transformer → Prediction Heads → SE(3) 校正矩阵。 每一步都伴随着明确的 tensor shape 变换，理解这些变换是调试和优化的基础。

📁

Calib 文件解析

Custom:
camera_intrinsic → intrinsic (3×3)

gt_T ∈ SE(3): (4, 4)
Phase 2: Dataset & collate_fn — 数据预处理
⚙

collate_fn — 从单样本到 batch 的完整变换链

1
Dataset.__getitem__()
读取 image, pointcloud, calib → 单样本 dict
img: (H_orig, W_orig, 3)   pc: (N, 3/4)
2
crop_and_resize(img)
统一图像尺寸 (center crop + resize)
→ (target_H, target_W, 3) e.g. (360, 640, 3)
3
intrinsic 缩放
fx, fy, cx, cy 按 resize 比例缩放
intrinsic: (3, 3) (值已调整)
4
ego_filter
|x|>3m & |y|>3m 过滤自车附近点
pc: (N', 3) where N' < N
5
range_filter
xbound/ybound/zbound 裁剪到 BEV 范围
pc: (N'', 3) where N'' < N'
6
perturbation → init_T
gt_T + noise (truncated_normal/uniform, per_axis)
init_T: (4, 4) SE(3)
7
PC padding + mask
batch 内 pad 到 max_N, 生成 mask
pc: (B, max_N, 3)   mask: (B, max_N)
8
to_tensor + to_device
numpy → torch.Tensor → GPU
img: (B,3,H,W)   pc: (B,N,3)   T: (B,4,4)
关键变换: collate_fn 是数据进入模型前的最后一道关卡。 ego_filter 和 range_filter 直接影响点云利用率 — 过滤太多会丢失信息，过滤太少会引入噪声。 PC padding 使用 mask 确保 padding 点不参与计算。
Phase 3: Image Branch (Cam2BEV) — 图像到 BEV 的投影
📷

Image Branch 完整 tensor 变换链

1
SwinT-Tiny (backbone)
预训练 ImageNet, backbone_lr × 0.1
(B,3,H,W) → multi-scale features
2
FPN 多尺度融合
合并不同分辨率特征
→ (B, 1, C_neck, H/8, W/8)
3
LSS 深度估计
D=89 bins, 每像素预测深度分布
→ (B, 1, C, D=89, H/8, W/8)
4
Frustum 几何变换
cam2ego_T (依赖 init_T!) 投影到 ego 坐标
→ 3D frustum points in ego frame
5
BEV Pool CUDA
QuickCumsum scatter 到 BEV 网格
→ (B, C×nZ, nX, nY)
6
ProjectionHead
Linear → GELU → Linear → LayerNorm
→ (B, 128, 100, 100)
init_T 敏感性: Step 4 的 Frustum 变换使用 init_T 的逆矩阵。 当 init_T 偏差 >5° 时，BEV 特征会严重错位，导致与 PC Branch 的 BEV 特征无法对齐。 这是大扰动场景下精度下降的根本原因。
Phase 4: PC Branch (Lidar2BEV) — 点云到 BEV 的投影
☁

PC Branch 完整 tensor 变换链

1
PointToVoxel (spconv)
vsize 由 bev_settings 计算, max 120k voxels
(B, N, 3) → SparseConvTensor
2
SparseEncoder Layer 1
SparseBasicBlock, 3D sparse conv
→ 16 channels
3
SparseEncoder Layer 2
下采样 + 残差
→ 32 channels
4
SparseEncoder Layer 3
下采样 + 残差
→ 64 channels
5
SparseEncoder Layer 4
下采样 + 残差
→ 128 channels
6
Z 压缩 + dense()
沿 Z 轴压缩, sparse → dense
→ (B, C×Z, nX, nY)
7
ProjectionHead
Linear → GELU → Linear → LayerNorm
→ (B, 128, 100, 100)
稳定锚点: 整个 PC Branch 不依赖任何标定参数 — 点云直接在 LiDAR 坐标系体素化。 这意味着无论 init_T 偏差多大，PC Branch 的 BEV 特征始终是稳定的空间参考。 Fusion 模块需要学会利用这个"地基"来校正 Image Branch 的偏移。
Phase 5: Fusion → Transformer → Output — 从 BEV 特征到标定结果
⚡

融合、推理、输出的完整变换链

1
ConvFuser (concat + 1×1 Conv)
concat(img_bev, pc_bev) → Conv → BN → ReLU
(B,128,100,100) + (B,128,100,100) → (B,256,100,100) → (B,embed_dim,H,W)
2
+ pose_embed
可学习位置编码, 注入空间先验
(B, embed_dim, H, W) + (1, embed_dim, H, W)
3
Deformable Transformer × L layers
DeformableAttention + MLP + DropPath(0.1)
(B, embed_dim, H, W) → (B, embed_dim, H, W)
4
Global Mean Pool
空间维度压缩为全局特征
(B, embed_dim, H, W) → (B, embed_dim)
5
head_dropout(0.1)
正则化, 防止过拟合
(B, embed_dim) → (B, embed_dim)
6
Translation Head
Linear(embed_dim → 3)
→ ΔT ∈ ℝ³ (B, 3)
7
Rotation Head
Linear(embed_dim → 4)
→ Δq ∈ ℝ⁴ (B, 4) quaternion
8
get_T_matrix()
quat→mat + tvec→mat → SE(3) 变换矩阵
→ pred_T (B, 4, 4)
9
校正计算
T_gt = T_pred⁻¹ · T_init
→ calibrated_T (B, 4, 4)
Phase 6: Loss Computation — 损失计算数据流
📈

Loss 输入数据流

a
translation_loss
SmoothL1(ΔT, gt_ΔT) → scalar
b
rotation_loss
geodesic(Δq, gt_Δq) → scalar
c
PC_reproj_loss
‖T⁻¹·T̂·p − p‖ → scalar
d
quat_norm_loss
(‖q‖²−1)² → scalar
e
AdaptiveAxisRotation
euler_errors × axis_weights → scalar
💡

Loss 聚合与反向传播

total_loss =
  1.0 × translation_loss
  + 0.5 × rotation_loss
  + 0.5 × PC_reproj_loss
  + 0.5 × quat_norm_loss
  + 0.3 × axis_loss
    (Roll×3 + Pitch×1.5 + Yaw×1)
  + 0.3 × tail_penalty
    (mean(topk(errors, 10%)) × 2.0)
反向传播路径:
total_loss → scaler.scale() → .backward()
→ 梯度流经: Heads → TR → Fuser → 双分支
→ scaler.unscale_() → clip(35.0) → step
Phase 7: Evaluation — 评估数据流
📊

从 pred_T 到物理误差指标的变换

输入
pred_T (B, 4, 4)
gt_T (B, 4, 4)
→
evaluate_sensor_extrinsic()
error_T = gt_T⁻¹ · pred_T
R = error_T[:3,:3]
t = error_T[:3,3]
euler = R → (roll, pitch, yaw)
→
输出: 8 维物理误差
旋转: roll(°), pitch(°), yaw(°)
平移: fwd(m), lat(m), ht(m)
总计: rot_error(°), trans_error(m)
LiDAR 坐标系: 误差分解在 LiDAR 坐标系下进行，物理意义明确。 fwd = 前进方向, lat = 侧向, ht = 高度。这比直接看 4×4 矩阵差异更直观。

Shape Summary — 完整 Tensor Shape 变换总结
💡

端到端 Shape 变换一览

Image Path:
(B,3,360,640) → SwinT → FPN → (B,1,256,45,80)
→ LSS → (B,1,128,89,45,80) → Frustum → BEV Pool
→ (B,C×nZ,100,100) → ProjHead → (B,128,100,100)
PC Path:
(B,N,3) → Voxel → SparseConvTensor
→ Sparse3D (16→32→64→128) → Z compress
→ (B,C×Z,nX,nY) → ProjHead → (B,128,100,100)
Fusion Path:
concat → (B,256,100,100) → Conv1x1 → (B,128,100,100) → +pose_embed → TR × L → mean_pool → (B,128) → Heads → (B,3) + (B,4) → get_T → (B,4,4)
