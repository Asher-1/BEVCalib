# BEVCalib 训练和测试流程文档

## 1. 环境准备

### 1.1 系统要求

- **OS**: Linux (Ubuntu 18.04+)
- **Python**: 3.11
- **CUDA**: 11.8
- **GPU**: NVIDIA GPU（至少 8GB 显存）

### 1.2 依赖安装

```bash
# 创建环境
conda create -n bevcalib python=3.11
conda activate bevcalib

# 安装 CUDA
conda install -c "nvidia/label/cuda-11.8.0" cuda-toolkit

# 安装 PyTorch
pip install torch==2.0.1 --extra-index-url https://download.pytorch.org/whl/cu118

# 安装依赖
pip install -r requirements.txt

# 编译 CUDA 扩展（必需）
cd kitti-bev-calib/img_branch/bev_pool && python setup.py build_ext --inplace
```

## 2. 训练脚本链

BEVCalib 使用分层脚本链管理训练：

```
configs/*.yaml → batch_train.sh → start_training.sh → train_universal.sh → train_kitti.py
```

### 2.1 YAML 配置驱动（推荐）

通过 YAML 文件定义实验，`batch_train.sh` 自动解析并顺序执行：

```bash
bash batch_train.sh configs/batch8_train_b26a_optim_v1.yaml
```

YAML 配置结构：

```yaml
global:
  dry_run: false
  batch_log_dir: "logs"

defaults:
  env:
    BEV_ZBOUND_STEP: 4.0
  params:
    angle_range_deg: 5
    trans_range: 0.15
    batch_size: 8
    learning_rate: 3e-4
    use_ddp: true
    ddp_gpus: 8
    rotation_only: true
    enable_axis_loss: true
    weight_axis_rotation: 0.3
    lr_schedule: cosine_warm_restarts
    warmup_epochs: 3
    backbone_lr_scale: 0.1
    drop_path_rate: 0.1
    head_dropout: 0.1
    perturb_distribution: truncated_normal
    per_axis_prob: 0.3
    per_axis_weights: "3.0,1.5,1.0"
    augment_pc_jitter: 0.01
    augment_pc_dropout: 0.05
    early_stopping_patience: 15

experiments:
  - name: "exp_v7_full"
    dataset: "B26A"
    version: "v7_opt_full"
  - name: "exp_v7_cosine"
    params:
      lr_schedule: cosine_warm_restarts
      warmup_epochs: 5
```

### 2.2 快速启动

```bash
# B26A 数据集，DDP 模式
bash start_training.sh B26A v1 --ddp

# 全量数据集
bash start_training.sh all v1 --ddp
```

### 2.3 通用训练脚本

`train_universal.sh` 支持三种模式：

| 模式 | 命令 | 说明 |
|------|------|------|
| scratch | `bash train_universal.sh scratch ...` | 从零训练，lr=1e-4，400 epochs |
| finetune | `bash train_universal.sh finetune ...` | 微调预训练模型，lr=4e-5，50 epochs |
| resume | `bash train_universal.sh resume ...` | 恢复中断训练 |

### 2.4 直接调用 Python

```bash
python kitti-bev-calib/train_kitti.py \
    --dataset_root /path/to/data \
    --use_custom_dataset \
    --log_dir ./logs/exp1 \
    --num_epochs 400 \
    --batch_size 8 \
    --lr 3e-4 \
    --angle_range_deg 5 \
    --trans_range 0.15 \
    --rotation_only \
    --enable_axis_loss \
    --lr_schedule cosine_warm_restarts \
    --warmup_epochs 3 \
    --backbone_lr_scale 0.1 \
    --drop_path_rate 0.1 \
    --head_dropout 0.1 \
    --perturb_distribution truncated_normal \
    --per_axis_prob 0.3 \
    --early_stopping_patience 15
```

## 3. 训练参数

### 3.1 模型参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--deformable` | 1 | 使用可变形注意力 |
| `--bev_encoder` | 0 | 使用 BEV 编码器 |
| `--rotation_only` | false | 仅预测旋转 |
| `--drop_path_rate` | 0.1 | DropPath 率 |
| `--head_dropout` | 0.1 | 预测头 Dropout |

### 3.2 损失参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--enable_axis_loss` | false | 启用逐轴旋转损失 |
| `--weight_axis_rotation` | 0.3 | 轴损失权重 |
| `--axis_weights` | 3.0,1.5,1.0 | Roll/Pitch/Yaw 权重 |
| `--tail_topk_ratio` | 0.1 | Tail penalty top-k 比例 |
| `--tail_weight` | 0.5 | Tail penalty 权重 |

### 3.3 优化参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--lr` | 1e-4 | 基础学习率 |
| `--wd` | 1e-4 | 权重衰减 |
| `--lr_schedule` | step | 调度器类型（step/cosine_warm_restarts） |
| `--warmup_epochs` | 0 | Warmup 轮数 |
| `--backbone_lr_scale` | 0.1 | Backbone 学习率倍率 |
| `--early_stopping_patience` | 0 | Early stopping（0=禁用） |

### 3.4 扰动参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--angle_range_deg` | 20 | 旋转扰动范围（度） |
| `--trans_range` | 1.5 | 平移扰动范围（米） |
| `--perturb_distribution` | uniform | 分布（uniform/truncated_normal） |
| `--per_axis_prob` | 0.0 | 单轴扰动概率 |
| `--per_axis_weights` | None | 轴采样权重 |

### 3.5 增强参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--augment_pc_jitter` | 0.0 | 点云高斯抖动 σ |
| `--augment_pc_dropout` | 0.0 | 点云随机丢弃率 |
| `--augment_color_jitter` | 0.0 | 图像颜色抖动强度 |

## 4. 分布式训练（DDP）

### 4.1 单机多卡

```bash
bash start_training.sh B26A v1 --ddp
```

`train_universal.sh` 自动使用 `torchrun --standalone --nproc_per_node=N`。

### 4.2 多机多卡

```bash
bash train_universal.sh scratch \
    --ddp 8 \
    --nnodes 4 \
    --node_rank $RANK \
    --master_addr $MASTER_ADDR \
    --master_port $MASTER_PORT
```

### 4.3 DDP 行为

| 角色 | 行为 |
|------|------|
| Master (Rank 0) | 日志写入、TensorBoard、检查点保存、验证评估 |
| Workers (Rank 1~N) | 仅前向+反向，stdout 静默，梯度通过 NCCL all-reduce 同步 |

## 5. 训练监控

### 5.1 日志系统

统一日志架构：所有输出汇聚到 `train.log`

- **Config Box**：Shell 脚本参数记录
- **tprint()**：带时间戳的 stdout + 文件写入
- **_StderrTee**：stderr 拦截到日志
- **capture_prints**：第三方库输出捕获

### 5.2 TensorBoard

```bash
tensorboard --logdir ./logs
```

记录指标：train_loss, val_loss, rot_error, trans_error, per-axis R/P/Y, P50/P90/P99/Max, learning_rate

### 5.3 检查点策略

| 文件 | 触发条件 |
|------|----------|
| `ckpt_best_val.pth` | 验证损失最优 |
| `ckpt_{epoch}.pth` | 每 N epochs |
| `latest_checkpoint.pth` | 用于恢复训练 |
| `train_kitti_src.py` | 源代码快照 |

检查点内容：`model_state_dict`, `optimizer_state_dict`, `scheduler_state`, `scaler_state`, `epoch`, `best_val_loss`, `args`

## 6. 推理与评估

### 6.1 推理命令

```bash
python kitti-bev-calib/inference_kitti.py \
    --dataset_root /path/to/data \
    --ckpt_path ./logs/exp1/ckpt_best_val.pth \
    --angle_range_deg 5 \
    --trans_range 0.15 \
    --batch_size 1
```

### 6.2 检查点评估

```bash
python evaluate_checkpoint.py \
    --ckpt_path ./logs/exp1/ckpt_best_val.pth \
    --dataset_root /path/to/data
```

### 6.3 评估指标

| 指标 | 说明 | 优秀标准 |
|------|------|----------|
| 旋转误差 | Roll/Pitch/Yaw（度） | < 0.5° |
| 平移误差 | X/Y/Z（米） | < 0.05m |
| 像素位移 | 投影点偏移 | < 1px |
| 改善百分比 | 误差降低比例 | > 90% |

### 6.4 模型导出

```
best_model.pth → bevcalib_inference.py → torch2drinfer.py → drinfer_infer.py → Production
```

## 7. 常见问题

### CUDA out of memory

- 减小 `--batch_size`
- 减小 BEV 空间范围（调整 `BEV_ZBOUND_STEP`）
- 确认 AMP 已启用

### 损失不下降

- 检查学习率和调度器配置
- 尝试 `truncated_normal` 分布
- 启用 `--enable_axis_loss`
- 检查数据加载是否正确

### BEV Pool 编译错误

```bash
cd kitti-bev-calib/img_branch/bev_pool && python setup.py build_ext --inplace
```

确保 CUDA 版本与 PyTorch 匹配。
